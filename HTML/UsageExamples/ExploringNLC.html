Understanding Image Outputs

When Land Drill generates images, it always generates two copies. One copy uses the true pixel values and is intended for use with geo-software and analyses. The second copy uses pixels values that are scaled between 0 and 255. The scaled, or normalized images, can be displayed with any photo viewer and easily incoprtated into documents and presentations.

Images Generated by Land Drill (.tiff)

  1. Core Count Image (Real32)
    2. An image where each pixel's value is equal to the number of times that pixel's core sample appears throughout the scene.
  2. State Count Image (Byte)
    3. An image where each pixel's value is equal to the number of distinct land cover types found within the core sample that was extracted from that pixel's location. Any state count can be derrived from the State Count Image.
  3. Single State Core Sample Image (Boolean, Byte)
    4. A Boolean image that marks the location of land cover persistence.
  4. Multi-State Core Sample Image (Boolean, Byte)
    5. A Boolean image that marks the location of pixels that experience any land cover change.
  5. Maximum State Core Sample Image (Boolean, Byte)
    6. A Boolean image that marks the location of pixels that had core samples with the highest number of distinct land cover types.

Normalized Core Count Image

The brighter a pixel is in the Core Count Image, the more times its core sample appears throughout the image. Many times, the brightest pixels correspond to land cover persistence. When viewed within geo-software and colorized with a palette, the core count image may better reveal where identical cores exist.

Normalized Multi-State Core Sample Image

Land Drill will also produce a pair of images that identify core samples that experienced any amount changed.

Likewise, another pair of images are produced to identify all core samples that did not experience any change.

Normalized Single-State Core Sample Image


Any other state count of interest can be derrived from the State Count Image. Pixel values are equal to the number of distinct land cover types found within each core sample.

Core Sample Locator

The Core Sample Locator tool can be used to identify specific core samples or transitions. Any core sample can be located. The Pixel History Report can help identify interesting core samples to locate.
